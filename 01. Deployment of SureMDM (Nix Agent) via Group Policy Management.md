## 1. 🎯 Purpose
This document outlines the procedure to deploy the **SureMDM (Nix Agent)** executable file along with its configuration setup using:

- Active Directory (AD)  
- Group Policy Management (GPO)  

on **Windows Server 2022 Standard Evaluation**.

---

## 2. 📌 Scope
This SOP applies to:

- 👨‍💻 IT Administrators managing Active Directory environments  
- 🖥️ Deployment of software across domain-joined computers  
- 🏢 Systems operating under Windows Server 2022  

---

## 3. 📖 Definitions

- **GPO (Group Policy Object):** A feature in Active Directory used to control settings across multiple computers.  
- **SureMDM / Nix Agent:** Device management software used for remote management and enrollment.  
- **SysVol:** A shared directory that stores server copies of public files in a domain.  
- **RDP (Remote Desktop Protocol):** A protocol used to remotely access another computer.  

---

## 4. ⚙️ Procedure / Step-by-Step Instructions

### 4.1 🖥️ Access Remote Server via RDP
1. Open **Run dialog**:
```

Win + R

```
2. Type:
```

mstsc /admin

```
3. Click **OK**  
4. In Remote Desktop Connection:
- Enter IP Address (e.g., `192.168.1.11`)  
5. Click **Connect**  

---

### 4.2 🔐 Login to Server
A user credential prompt will appear.

**Enter credentials:**

| Username       | Password   |
|---------------|------------|
| Administrator | ********** |

📌 *Table 1: User Credential*

- Click **OK**

---

### 4.3 🧭 Open Group Policy Management
- The AD Server interface will open  
- Open **Run dialog**:
```

Win + R

```
- Type:
```

gpmc.msc

```
- Click **OK**

---

### 4.4 📂 Locate the GPO
- Navigate to:
```

Domain > xitiztechservice.local > Domain_Computers

```
- Locate the GPO:
```

Deploy Nixagent EXE File

```

---

### 4.5 ✏️ Edit the GPO
1. Right-click on:
```

Deploy Nixagent EXE File

```
2. Click **Edit**

---

### 4.6 📜 Navigate to Startup Scripts
Follow the path:

```

Computer Configuration > Policies > Windows Settings > Scripts (Startup/Shutdown) > Startup > Scripts

```

---

### 4.7 🚀 Configure Script Deployment
1. **Script Name:** `install_nixagent.cmd`  
2. **Purpose:**  
   - Push the executable file from Active Directory server to domain computers  

---

### 4.8 📁 Locate Script File Path
1. Click **Edit**  
2. Click **Browse**

📍 **Script Location:**
```

\xitiztechservice.local\SysVol\xitiztechservice.local\Policies{6DC4AF6E-3587-4435-916A-E5980514C259}\Machine\Scripts\Startup

````

📌 **Note:**
- This path is auto-generated when creating a GPO  
- Custom scripts should be placed in this directory  

---

### 4.9 📝 View/Edit Script
- Open the script using **Notepad**

**Script Content:**
```cmd
@echo off
echo NixAgent install started >> C:\nixagent_install.log

if exist "C:\Program Files\NixAgent\nixagent.exe" (
echo Already installed >> C:\nixagent_install.log
exit /b 0
)

\\ADDC01\ADSharingFolder$\nixagent_Setup_x64.exe ^
/VERYSILENT ^
/SUPPRESSMSGBOXES ^
/NORESTART ^
/NIXCLI eyJhY2NvdW50X2lkIjoiMDIyNjA8888887675655hbmdyb3VwLmluLnN1cmV654345679tZSIsInBhc3N3b3JkIjoiIiwiZGV2aWNlX25hbWVfdHlwZSI6M30=

echo Install command executed >> C:\nixagent_install.log
exit /b 0
````

## 5. 🔍 Explanation of Script Components

### 5.1 ⚡ Installer Execution

```
nixagent_Setup_x64.exe /VERYSILENT
```

* Runs installer in silent mode
* No user interaction required
* No pop-ups or prompts displayed

---

### 5.2 ⚙️ Command Parameters

* `/VERYSILENT` → Silent installation
* `/SUPPRESSMSGBOXES` → Suppresses message boxes
* `/NORESTART` → Prevents automatic system restart

---

### 5.3 🔐 Encoded Configuration (/NIXCLI)

```
eyJhY2NvdW50X2lkIjoiMDIyNjA8888887675655hbmdyb3VwLmluLnN1cmV654345679tZSIsInBhc3N3b3JkIjoiIiwiZGV2aWNlX25hbWVfdHlwZSI6M30=
```

📌 This is a **Base64 encoded JSON** containing:

* Account ID
* SureMDM Server URL
* Device Group Path
* Enrollment Settings

---

## 6. 🔑 Encryption Key

### 6.1 📘 Encryption Key Description

* Generated from the SureMDM dashboard
* Specific to the **Root Organizational Unit (OU)**
* Separate keys required for different OUs
* Must replace previous key when updated

---

### 6.2 🛠️ Generate Encryption Key

Access the SureMDM Portal:
👉 (https://xitiztechservice.in.suremdm.io/)

---

### 6.2.1 📋 Procedure

1. Navigate to:

   ```
   Enrollment > Manual Enrollment > Windows > CLI Based Enrollment
   ```
2. Click **Configure**
3. Default OU: `KTM`
4. Click folder icon to change OU
5. Select required OU (e.g., `IT_Team`)
6. Click **OK**
7. Verify OU change
8. Click **Apply**

---

### 6.2.2 ✅ Result

* A new encryption key is generated
* Key differs per OU

📌 **Note:**

* Use OU-specific keys in deployment scripts
* Ensures proper device enrollment and policy mapping

---

### 6.3 🔄 Update Procedure

* Locate configuration string:

```
/NIXCLI eyJhY2NvdW50X2lkIjoiMDIyNjA8888887675655hbmdyb3VwLmluLnN1cmV654345679tZSIsInBhc3N3b3JkIjoiIiwiZGV2aWNlX25hbWVfdHlwZSI6M30=
```

* Replace **only** the encryption key portion after `/NIXCLI`

⚠️ **Important:**

* Do NOT modify other parameters
* Incorrect edits may cause configuration failure

## 7. 📂 Location of EXE File in the Shared Folder

### 📍 File Location

* Navigate to:

  ```
  C:\
  ```
* Open shared folder:

  ```
  ADSharingFolder
  ```

### 📄 Details

* Contains:

  ```
  nixagent_Setup_x64.exe
  ```

### 🎯 Purpose

* Used to install SureMDM agent on domain computers

---

### 📌 Notes

* Ensure proper **sharing and NTFS permissions**
* Verify network path:

  ```
  \\ADDC01\ADSharingFolder\nixagent_Setup_x64.exe
  ```
* File must be accessible from client systems
* One EXE can serve all domain computers
* OU-specific EXE files may also be used
* EXE files can be downloaded from SureMDM portal

📌 If using Root (Home) EXE:

* Devices initially enroll under default OU
* Can be reassigned via SureMDM portal

---

## 8. 📏 Policies / Guidelines

* Ensure all domain computers have access to:

  ```
  \\ADDC01\ADSharingFolder$
  ```
* Verify script execution permissions
* Ensure GPO is linked to correct OU
* Always test deployment in a **controlled environment** before production rollout
